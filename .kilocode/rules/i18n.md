# Internationalization (i18n)

---
trigger: always_on
---

## Guidelines

- All user-facing text MUST support internationalization (i18n).
- Use the `@grammyjs/i18n` library for Telegram Bot text.
- Localization files should be stored in the `locales/` directory using the Fluent (`.ftl`) format.
- Avoid hardcoding strings in the source code. Use `ctx.t('key')` or equivalent methods to retrieve translated strings.
- When adding new features or modifying existing ones, ensure all new text is added to the relevant `.ftl` files.
- When passing numeric IDs (like User ID or Artwork ID) to Fluent/i18n functions, ALWAYS convert them to strings explicitly (e.g., `String(id)`). Fluent automatically formats numbers (e.g., adds commas), which is undesirable for IDs.
